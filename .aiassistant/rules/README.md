# 규칙 문서 구조

프로젝트의 코딩 규칙과 가이드라인을 체계적으로 관리합니다.

## 구조

- `foundations/`: 코드 품질의 근본 원칙 (응집도, 결합도, 가독성, 예측가능성)
- `conventions/`: 구체적인 코딩 컨벤션 (네이밍, 스타일, import)
- `architecture/`: 구조적 설계 원칙 (폴더 구조, 상태관리, API)
- `workflow/`: 협업 프로세스 (Git 워크플로우)

## 문서 추가 방식

### 1. 메타 태그 추가

문서 첫 줄에 추가:

```yaml
---
visibility: ai-only | human+ai
origin: cursor | assistant
---
```

### 2. 작성 원칙

- **간결함**: 50줄 미만 유지, 핵심만 포함
- **분리**: 큰 개념은 여러 규칙으로 나누기
- **명확성**: "해야 한다", "하지 않는다" 등 명확한 표현
- **예시**: 구체적 예시와 체크리스트 제공
- **상호 참조**: `(naming-conventions.md 참고)` 형식으로 참조

### 3. 구조 템플릿

```markdown
# 규칙 제목

## 개요

- 핵심 원칙 요약

## 세부 지침

- 구체적인 규칙 설명

## 체크리스트

- [ ] 검증 항목
```

## 태그 요약

| 태그                   | 의미                       |
| ---------------------- | -------------------------- |
| `visibility: human+ai` | 사람·AI 모두 참조          |
| `visibility: ai-only`  | AI만 참조                  |
| `origin: cursor`       | 범용 원칙 (foundations/)   |
| `origin: assistant`    | 특화 규칙 (conventions 등) |

## 사용 가이드

- **AI**: 작업 전 관련 규칙 확인, `foundations/` 원칙 항상 고려
- **사람**: 새 규칙 추가 시 적절한 디렉토리 선택, 메타 태그와 작성 원칙 준수
